# 🏢 نظام إدارة العمارات - Building Management System

نظام متكامل لإدارة العمارات السكنية مع نظام اشتراكات شهرية متطور ونظام توزيع المصروفات

---

## 🌟 المميزات الرئيسية

### 1. إدارة متعددة المستويات (Multi-Tenant)
- **مدير النظام**: إدارة جميع العمارات
- **مدير العمارة**: إدارة عمارة واحدة محددة
- **الساكن**: عرض بياناته الخاصة فقط

### 2. نظام الاشتراكات الشهرية 💰
- إنشاء أنواع اشتراكات متعددة (مصعد، حدائق، نظافة، قمامة)
- توليد تلقائي للاشتراكات الشهرية
- صفحة متخصصة لتسجيل الدفعات
- فلترة متقدمة (سنة، شهر، نوع الاشتراك)
- إحصائيات فورية (مدفوع، معلق، متأخر)
- زر تذكير يدوي للمتأخرين عن الدفع

### 3. نظام توزيع المصروفات 📊
- توزيع المصروفات على جميع السكان بالتساوي
- تحميل المصروف على ساكن محدد
- حساب تلقائي للحصص
- تتبع حالة الدفع لكل حصة
- صفحة مخصصة لعرض توزيع كل مصروف

### 4. نظام الإشعارات 🔔
- إشعار تلقائي عند إنشاء اشتراك جديد
- إشعار عند تسجيل الدفع
- إشعار عند توزيع مصروف جديد
- تذكيرات يدوية للمتأخرين
- عداد للإشعارات غير المقروءة

### 5. إدارة الشقق 🏠
- إضافة وتعديل بيانات الشقق
- ربط الشقق بالسكان
- متابعة حالة كل شقة

### 6. طلبات الصيانة 🔧
- تقديم طلبات صيانة من السكان
- متابعة حالة الطلبات
- إشعارات عند تغيير الحالة

### 7. لوحات تحكم متخصصة 📈
- لوحة تحكم للمدير العام
- لوحة تحكم لمدير العمارة
- لوحة تحكم للساكن
- إحصائيات شاملة ورسوم بيانية

## 📦 التثبيت

### المتطلبات:
- PHP >= 8.2
- Composer
- MySQL >= 8.0
- Node.js & NPM (اختياري)

### خطوات التثبيت:

1. **استنساخ المشروع**:
```bash
git clone <repository-url>
cd building-Management
```

2. **تثبيت المكتبات**:
```bash
composer install
```

3. **إعداد ملف البيئة**:
```bash
cp .env.example .env
php artisan key:generate
```

4. **إعداد قاعدة البيانات**:
```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=building_management
DB_USERNAME=root
DB_PASSWORD=
```

5. **تشغيل Migrations**:
```bash
php artisan migrate
```

6. **تشغيل Seeders (اختياري)**:
```bash
php artisan db:seed
```

7. **تشغيل المشروع**:
```bash
php artisan serve
```

8. **الوصول للنظام**:
```
http://localhost:8000
```

## 🎯 سيناريوهات الاستخدام

### سيناريو 1: إعداد النظام لأول مرة

1. تسجيل الدخول كمدير عمارة
2. إضافة الشقق من صفحة "الشقق"
3. إضافة المستخدمين (السكان) من صفحة "المستخدمين"
4. ربط كل ساكن بشقته
5. إنشاء أنواع الاشتراكات من صفحة "أنواع الاشتراكات"
6. تشغيل أمر توليد الاشتراكات

### سيناريو 2: تسجيل دفعات الاشتراكات الشهرية

1. الذهاب إلى "تسجيل الدفعات"
2. اختيار الشهر والسنة
3. اختيار نوع الاشتراك (اختياري)
4. الضغط على "تطبيق"
5. تسجيل الدفع لكل ساكن قام بالدفع
6. إرسال تذكير للمتأخرين (زر "إرسال تذكير")

### سيناريو 3: إضافة مصروف وتوزيعه

1. الذهاب إلى "المصروفات"
2. إضافة مصروف جديد
3. اختيار نوع التوزيع:
   - **جميع السكان**: سيتم التقسيم بالتساوي
   - **شقة محددة**: اختيار الشقة المستهدفة
4. حفظ المصروف
5. الذهاب إلى "عرض الحصص" لرؤية التوزيع
6. تسجيل الدفع لكل حصة

## 🤖 الأوامر (Commands)

### توليد الاشتراكات الشهرية:
```bash
# توليد للشهر الحالي
php artisan subscriptions:generate-monthly

# توليد لشهر محدد
php artisan subscriptions:generate-monthly --month=3 --year=2026
```

### جدولة التوليد التلقائي:
في ملف `app/Console/Kernel.php`:
```php
protected function schedule(Schedule $schedule)
{
    $schedule->command('subscriptions:generate-monthly')
             ->monthlyOn(1, '00:00');
}
```

تفعيل Scheduler في Cron:
```bash
* * * * * cd /path-to-project && php artisan schedule:run >> /dev/null 2>&1
```

## 🐛 استكشاف الأخطاء الشائعة

### خطأ: Class 'Illuminate\Support\Facades\Schema' not found
**الحل**: تأكد من تشغيل `composer install`

### خطأ: SQLSTATE[HY000] [2002] Connection refused
**الحل**: تأكد من تشغيل MySQL وصحة بيانات الاتصال في `.env`

### خطأ: Route [login] not defined
**الحل**: تأكد من وجود `require __DIR__.'/auth.php';` في `routes/web.php`

### الإشعارات لا تعمل
**الحل**: تحقق من وجود علاقة `resident()` في نموذج `Apartment`
